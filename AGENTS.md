# 상세페이지 생성 도구 작업 가이드

이 프로젝트는 개발을 모르는 사람도 웹에서 상품 상세페이지를 만들 수 있게 해주는 도구입니다.  
터미널에서 쓰던 상세페이지 생성 흐름을 웹 화면으로 옮긴 것입니다.

## 한 줄 요약

사용자는 브랜드 기본값과 상품 정보를 입력하고, 초안을 확인한 뒤, 승인된 초안으로 상세페이지 이미지를 생성합니다.

## 주요 화면

- `/login`: 로그인 화면입니다. 로그인 전에는 왼쪽 메뉴를 보여주지 않습니다.
- `/brands`: 브랜드 기본값을 등록, 수정, 삭제합니다.
- `/detail-pages/new`: 상품 정보를 입력하고 상세페이지 초안을 만듭니다.
- `/detail-pages/[id]/approval`: 상세페이지 초안을 확인하고 컷별로 수정합니다.
- `/detail-pages/[id]/review`: 생성된 상세페이지 이미지를 확인하고 수정 요청을 합니다.
- `/detail-pages`: 생성된 상세페이지 목록입니다.
- `/md-workspace`: 운영 메모리 MD 파일을 보고 수정합니다.
- `/admin/users`: 관리자만 볼 수 있는 사용자 관리 화면입니다.

## 기본 로그인 규칙

- 관리자 계정만 사용자 관리 메뉴를 볼 수 있습니다.
- 현재 MVP에서는 모든 사용자의 비밀번호를 `1234`로 맞춥니다.
- 실제 운영용 DB와 권한 구조는 나중에 붙입니다.

## 사용자가 보는 기본 흐름

1. 로그인합니다.
2. 브랜드 기본값을 등록합니다.
   - 브랜드명
   - 로고
   - 포인트 컬러
   - 기본 톤
   - 배송 안내
   - 반품/교환 안내
   - 선택 안내사항
3. 필요하면 메모리 관리에서 운영 메모리를 수정합니다.
4. 상세페이지 초안 만들기 화면에서 상품 정보를 입력합니다.
   - 상품명
   - 카테고리
   - 컷 수
   - 상품 사진
   - 썸네일 생성 여부
   - 판매 포인트
   - 상품 사실 정보
   - 상품별 안내사항
5. 상세페이지 초안 화면에서 컷별 초안을 확인합니다.
6. 초안을 직접 수정하거나 특정 컷만 다시 만듭니다.
7. 초안을 승인하면 상세페이지 이미지가 생성됩니다.
8. 이미지 생성 리뷰 화면에서 컷별 이미지를 확인합니다.
9. 필요한 컷만 수정 요청하거나 다시 생성합니다.
10. 개별 이미지를 다운로드하거나 전체 상세페이지를 확인합니다.

## 생성 프로세스

### 1. 브랜드 기본값 저장

브랜드 기본값은 사용자별로 저장됩니다.  
한 사용자는 최대 5개 브랜드까지 등록할 수 있습니다.

브랜드에 저장된 안내사항은 상품 상세페이지 생성 때 자동으로 불러옵니다.  
선택 안내사항은 전체 카테고리에 적용하거나, 특정 카테고리에만 적용할 수 있습니다.

### 2. 상품 초안 생성

상품 정보를 저장하면 승인용 MD가 먼저 만들어집니다.  
이미지를 바로 만들지 않고, 사람이 먼저 초안을 확인할 수 있게 하기 위한 단계입니다.

초안은 컷 단위로 나뉩니다.  
예를 들어 6컷 상세페이지라면 Cut 01부터 Cut 06까지 생성됩니다.

### 3. 초안 수정

사용자는 전체 초안을 다시 만들 수 있습니다.  
또는 특정 컷만 선택해서 다시 만들 수 있습니다.

특정 컷만 다시 만든 경우에는 그 컷만 승인하고, 그 컷 이미지만 다시 생성할 수 있어야 합니다.

### 4. 이미지 생성

승인된 MD를 기준으로 이미지를 생성합니다.  
이미지는 컷별로 생성되며, 서로 중복되는 내용은 최대한 줄여야 합니다.

상세페이지는 컷 하나가 완성본이 아니라, 여러 컷이 모여 하나의 상세페이지가 됩니다.  
따라서 같은 고객센터, 배송, 반품/교환 문구가 여러 컷에 반복되면 안 됩니다.

### 5. 이미지 수정 요청

이미지 리뷰 화면에서 컷별 수정 요청을 할 수 있습니다.

문구만 바꾸는 요청은 전체 이미지를 새로 기획하지 말고, 기존 컷 초안의 해당 문구를 바꾼 뒤 다시 생성하는 방향을 우선합니다.  
단, 로고 수정 요청은 로고를 AI가 새로 그리게 하지 말고 업로드된 원본 로고를 사용해야 합니다.

## 절대 지켜야 할 콘텐츠 규칙

- 사용자가 직접 입력한 배송 안내, 반품/교환 안내, 선택 안내사항은 누락하면 안 됩니다.
- 사용자가 입력한 안내사항의 의미를 임의로 바꾸면 안 됩니다.
- 사용자가 입력하지 않은 배송사, 기간, 고객센터 번호, 휴무일 같은 정보는 새로 만들어내면 안 됩니다.
- 확인되지 않은 효능, 인증, 수상, 판매량, 후기, 원산지, 성분 정보는 작성하면 안 됩니다.
- 정보가 부족하면 “확인 필요”로 남기는 것이 맞습니다.
- 고객센터, 배송, 반품/교환 안내는 일반 판매 컷에 반복하지 말고 안내 컷에 모읍니다.
- 안내사항의 텍스트는 보존하되, 디자인은 보기 좋게 바꿀 수 있습니다.

## 로고 규칙

- 브랜드 로고는 사용자가 업로드한 원본 파일을 최대한 그대로 사용합니다.
- AI가 로고를 새로 그리거나 심볼만 따로 만들어 쓰면 안 됩니다.
- 허용되는 변경은 필요한 경우 올블랙 또는 올화이트 처리 정도입니다.
- 상세페이지 이미지 생성 시 AI 프롬프트에는 로고를 직접 그리지 말라고 지시하고, 앱에서 원본 로고를 합성하는 방식을 우선합니다.

## 메모리 MD 규칙

- 메모리 관리 화면에서는 필요한 운영 메모리만 보여주고 수정합니다.
- skill 관련 MD는 사용자가 직접 수정하는 운영 메모리로 취급하지 않습니다.
- 메모리는 상세페이지 스타일, 금지 표현, 자주 쓰는 안내, 운영 기준을 보강하는 용도입니다.
- 메모리에 있는 내용은 초안과 이미지 프롬프트에 반영하되, 사용자 화면의 컷 항목에 불필요하게 길게 노출하지 않습니다.

## 데이터 저장 위치

- 임시 DB: `.data/detail-page-web-tool.json`
- 업로드 이미지와 생성 이미지: `.data/assets`
- 상품 사진: asset kind `product_photo`
- 브랜드 로고: asset kind `brand_logo`
- 생성된 컷 이미지: asset kind `generated_cut`
- 생성된 썸네일: asset kind `generated_thumbnail`

현재는 SaaS 운영 전 MVP라서 파일 기반 저장소를 사용합니다.  
실제 운영 DB, 마이그레이션, 테스트 데이터 초기화는 나중 단계에서 처리합니다.

## 중요한 코드 위치

- 서버 액션: `app/actions.ts`
- 브랜드 저장 로직: `lib/services/brands.ts`
- 상품 초안 저장 로직: `lib/services/product-drafts.ts`
- 승인용 MD 생성 로직: `lib/services/approval-md.ts`
- 이미지 생성 로직: `lib/services/image-generation.ts`
- 데이터 저장 유틸: `lib/store.ts`
- 초안 카드 UI: `components/ux/DraftReviewCards.tsx`
- 브랜드 기본값 미리보기: `components/ux/BrandDefaultsPreview.tsx`
- 상품 사진 업로드 UI: `components/ux/ProductPhotoUploadInput.tsx`
- 전체 스타일: `app/globals.css`

## 개발 명령어

```bash
npm.cmd run dev
npm.cmd run typecheck
npm.cmd run build
```

개발 서버는 기본적으로 `http://localhost:4174`를 사용합니다.

## 환경 변수

`.env.example` 기준으로 아래 값이 필요합니다.

```bash
OPENAI_API_KEY=
OPENAI_IMAGE_MODEL=gpt-image-2
OPENAI_IMAGE_QUALITY=medium
OPENAI_IMAGE_REVISION_QUALITY=high
```

이미지 품질을 올리고 싶으면 `OPENAI_IMAGE_QUALITY`를 `high`로 조정합니다.  
문구 수정 재생성 품질은 `OPENAI_IMAGE_REVISION_QUALITY`를 우선 확인합니다.

## 작업할 때 주의할 점

- 사용자 입력값을 보존하는 것이 가장 중요합니다.
- AI가 알아서 좋게 바꿔줄 것이라고 믿고 안내사항을 요약하면 안 됩니다.
- 한 컷만 고치는 요청이면 전체 상세페이지를 다시 흔들지 않습니다.
- 컷별 초안, 승인 MD, 생성 이미지의 관계를 깨뜨리지 않습니다.
- 숨겨둔 임시 코드나 사용하지 않는 액션을 남기지 않습니다.
- 수정 후에는 최소한 `npm.cmd run typecheck`를 실행합니다.
- 라우트나 서버 액션을 건드렸다면 `npm.cmd run build`까지 확인합니다.
